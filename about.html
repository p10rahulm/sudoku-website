Overview
The algorithm can be broken into three parts:
  • the state that's maintained
  • the update step when adding a new number
  • solving steps
We will go into each of these by parts
State
The state that we maintain consists of a
  • A Game Global
  • Sudoku Working Values
  • Process Next Queue
Game Global
Game global, which contains meta about what are the adjacent elements to any given element - in it's row, in it's column and in it's box.
Sudoku Working Values
Sudoku working values are maintained as numbers from 1 to 511 representing each of the 2^11 options that a single cell can take. Think of this as a 9 length array of the form [1,0,0,0,1,0,1,1,1] which means that the cell can take values 9,5,3,2,and 1. This state is translated in binary as 279 and this is the state of the cell. This is stored in an array. This makes it easier to perform operations as we use bitwise operators to remove and add elements to such arrays.
Process Next Queue
Our Process-Next list consists of 2 queues. One is an immediate process stack, where we have elements that can take up only 1 value. The other is a priority heap where we store the next value to iterate.
Updates
The update steps are different for when a new element is added and when we are solving. When we are solving, then when a new element is added, we add it to the to be processed list.
